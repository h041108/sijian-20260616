# Coze（扣子）3.0 vs 思见 分析

## Coze 是什么
字节跳动的 AI 智能体开发平台。零代码、拖拽式。2024年2月1日上线。

## Coze 核心能力（3.0，2026年6月）
- **智能体开发**：可视化 Prompt 编辑器，结构化人设+指令+约束
- **工作流**：拖拽 LLM/代码/条件/循环节点，搭建复杂业务逻辑
- **插件系统**：60+官方插件 + 自定义 API
- **知识库 RAG**：上传 PDF/DOC 私有数据
- **云电脑**（Ubuntu）：Agent 可自主写代码跑脚本
- **云手机**（Android 13）：Agent 自动操作 App（发帖、订酒店）
- **技能商店**：365+ 行业专业技能可复用
- **多 Agent 团队**：多人+多 Agent 协同，项目空间管理
- **视频创作**：基于 Seedance 2.0 的剧本→分镜→剪辑→配音全流程
- **扣子编程**：自然语言编程构建网页/App/小程序（已开源 Apache 2.0）
- **本地 Agent 接入**：可接入 Claude Code、Codex CLI、OpenClaw

## Coze vs 思见

| 维度 | Coze | 思见 |
|------|------|------|
| 定位 | 开发平台（用户自行搭建Agent） | 产品（用户直接使用） |
| 目标用户 | 开发者/技术玩家 | 普通用户 |
| 使用门槛 | 需要"搭建"思维 | 一句话输入 |
| 核心优势 | 灵活、可组合、生态丰富 | **不需要搭建，直接出结果** |
| 弱点 | 用户不知道搭什么 | 功能深度不够 |

## 对思见的关键启示

1. **Coze 是平台不是产品** — 用户要自己搭建 Agent，99%的人不会
2. **思见的空间是做 Coze 用户不搭建的东西** — 把你的"副业引擎"做成 Coze 上 100 个插件都拼不出来的完整产品
3. **Coze 云手机能力是思见缺失的** — 自动发布到小红书/抖音需要这个
4. **技能商店模式值得借鉴** — 思见用户成功跑通一个赚钱模型后，可以封装为"技能模板"给其他用户用
5. **Coze 视频创作也用 Seedance 2.0** — 同一底层，思见要做的是更好的上层体验
