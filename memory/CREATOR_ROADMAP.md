# 思见垂直自动化运营智能体 · 实施路线图

## 零、产品定义

> **思见 Creator = 一句话开启自媒体赚钱之路**
>
> 对标 Polsia，但专注内容变现赛道。
> Polsia 说"AI替你开公司"，思见说"AI替你经营自媒体账号"。

## 二、分阶段实施计划

### Phase 0: 基础设施（3天）

**目标: 建好引擎加载系统**

```
新增文件:
  src/lib/creator-engine.ts        — 引擎注册中心 + 按需调取
  src/lib/engine-registry.ts        — 引擎元数据（15个引擎的触发条件/变量槽）
  src/app/api/creator/route.ts      — Creator API（引擎调用统一入口）

修改文件:
  src/app/page.tsx                  — 添加 Creator 入口按钮
```

**引擎调用协议:**
```typescript
// 按需调取引擎
const result = await dispatchEngine("E02-persona-modeling", {
  platform: "抖音",
  niche: "护肤品测评",
  targetAudience: "25-35岁女性"
})
// → 返回完整人设建模方案

// 引擎链式调用
const pipeline = createPipeline([
  "E02-persona-modeling",   // 1. 建人设
  "E03-competitor-analysis", // 2. 对标拆解  
  "E15-topic-generation",   // 3. 生成选题
  "E05-shot-director",      // 4. 分镜脚本
  "E04-video-prompt",       // 5. 视频提示词
])
// → 一键跑通选题→视频全流程
```

### Phase 1: 最小闭环（1周）

**目标: 用户选赛道 → 人设+ 选题 → 生成第一条视频**

```
Day 1-2: E02 人设建模引擎
  - 输入: 平台 + 赛道 + 目标人群
  - 输出: 人设方案（昵称/简介/视觉风格/内容方向）
  - API: POST /api/creator/persona

Day 3-4: E15 选题引擎 + E03 对标拆解
  - 输入: 赛道 + 平台
  - Tavily搜索 → DeepSeek拆解 → 输出10个选题
  - API: POST /api/creator/topics

Day 5-6: E05 + E04 → 即影视频生成
  - 选题 → 脚本 → 即影生成视频
  - API: POST /api/creator/generate-video

Day 7: CreatorDashboard 第一版
  - 三步向导页面: 选赛道 → 确认选题 → 生成视频
```

### Phase 2: 内容生产自动化（1周）

**目标: 日更型博主的日常流程自动化**

```
Day 8-9: 每日选题自动推送
  - 定时生成3个选题 + 标题
  - 用户确认/自动选 → 自动生成视频
  - API: POST /api/creator/daily-brief

Day 10-11: E06 BGM + E13 封面
  - 视频生成完自动配BGM
  - 自动生成3版封面供选择

Day 12-14: E12 内容复刻引擎
  - 对标爆款 → 差异化复刻 → 3版文案
  - 适合"看到别人爆了想跟"的场景
```

### Phase 3: 运营闭环（2周）

**目标: 发布+数据追踪+优化建议**

```
Day 15-18: E08 数据分析引擎
  - 用户输入平台数据 → AI分析 → 优化建议
  - 周报自动生成

Day 19-21: E13 评论运营 + E09 投流优化
  - 评论自动回复模板
  - 投流策略建议

Day 22-28: 自动发布（需要平台API）
  - 先做手动发布+数据追踪
  - 后续接入抖音/小红书开放平台
```

**毛利率:** 按创作者版 ¥29/月 + 10条视频 → 成本 ~¥15 → 毛利 ~¥14/用户/月

## 五、与 Polsia 的关键差异

| 维度 | Polsia | 思见 Creator |
|------|--------|-------------|
| 启动 | 输入创业想法 | 输入赛道 |
| 产出 | AI生成的网站 | 真实视频内容 |
| 用户看到的结果 | 网站+广告数据 | 视频+粉丝+收入 |
| 失败率 | 93.7% | 取决于内容质量 |
| 为什么更好的理由 | — | 内容质量天然高于代码质量 |
