## 1. 产品总览

### 1.1 一句话定义

> **即影 = 思见平台的 AI 全媒体生成中枢**——用户提供一句话、一张参考图或一个参考文案，由全球顶级多模态大模型驱动，自动完成从"创意构思→提示词工程→角色/场景建立→多格式生成→质量校验→成品交付"的完整创作闭环。

### 1.2 战略价值

```
思见平台 = 认知层（思维空间/思维线）              ← 理解用户
         ↓
         即影 = 创作层（图片/视频/漫剧/短剧）     ← 创造内容
         ↓
         商业层（15引擎/数据运营/变现）           ← 实现价值
```

即影是思见从"认知工具"升级为"创作平台"的战略性产品。如果品质不够顶，整个平台的竞争力就不可持续。

### 1.3 产品原则

| 原则 | 说明 |
|------|------|
| **轻参与，重品质** | 用户一句话启动，AI 主导全流程。但用户可选择性提供参考素材来提升品质 |
| **前置做足，后顾无忧** | 生成之前的准备工作（提示词/角色/场景/参考图）比生成本身更重要 |
| **全球顶级引擎** | 每个环节调用最适合的大模型，不自我设限，持续接入最新SOTA模型 |
| **连续性第一** | 多帧/多镜头必须保证角色、风格、场景的绝对一致 |
| **成品为王** | 所有生成结果可下载、可分享、可商用 |

## 3. 用户旅程

### 3.1 典型路径

```
用户进入即影
     │
     ├─ 选择产品类型: [图片] [视频]
     │
     ├─ 选择产品子类型: [封面] [海报] [详情页] [LOGO] ...
     │                    [短剧] [漫剧] [带货] [口播] [教育] ...
     │
     ├─ 提供核心输入（用户轻参与）
     │   ├─ 📝 一句话描述（必需）
     │   ├─ 🖼️ 参考图（可选 — 强烈建议提供）
     │   ├─ 📄 参考文案/链接（可选 — 爆款对标）
     │   ├─ 🏷️ LOGO/品牌素材（可选 — 涉及品牌时）
     │   └─ ⚙️ 高级参数（角色/风格/比例/时长等）
     │
     ├─ 【系统】自动执行前置质量保障（见第5章）
     │
     ├─ 【系统】调用AI引擎生成 → 用户实时预览进度
     │
     ├─ 用户审查结果
     │   ├─ ✅ 满意 → 下载/分享/发布
     │   └─ 🔄 不满意 → 局部修改/重新生成/调整参数
     │
     └─ 【系统】自动保存至作品集 + 数据追踪
```

### 3.2 用户参与度分级

| 层级 | 参与度 | 典型输入 | 品质预期 |
|------|--------|---------|---------|
| L1 | 极轻 | 一句话提示 | 80分 — 通用品质 |
| L2 | 轻度 | 一句话 + 1张参考图 | 85分 — 风格准确 |
| L3 | 适中 | 一句话 + 参考图 + 参考文案/对标链接 | 90分 — 爆款品质 |
| L4 | 深度 | L3 + 角色三视图 + 场景限定 + 产品多角度图 | 95分 — 商业级 |

## 4. 核心模型引擎

### 4.1 模型调度架构

```
即影模型路由
     │
     ├─ 📝 LLM/剧本组（故事/分镜/提示词）
     │   ├─ DeepSeek V3        — 中文故事+逻辑结构+分镜脚本（主）
     │   ├─ Claude 4           — 创意叙事+角色塑造+情感表达（备）
     │   └─ 通义千问/Qwen      — 中文适配+文化语境（备）
     │
     ├─ 🖼️ 图像生成组（图片/关键帧）
     │   ├─ Midjourney V7      — 艺术质感+光影构图（主-高端）
     │   ├─ 即梦/Jimeng        — 中式美学+角色一致性（主-高频）
     │   ├─ Flux/StableDiffusion3 — 快速迭代+开源可控（备）
     │   └─ DALL-E 3           — 精确指令跟随+文字渲染（备）
     │
     ├─ 🎬 视频生成组（动态镜头/动画）
     │   ├─ Seedance 2.0       — 文生视频+图生视频+动作流畅（主）
     │   ├─ 可灵/Kling         — 物理模拟+动作连贯（备）
     │   ├─ Runway Gen-4       — 电影级画质+运动模糊（备）
     │   ├─ Pika 2.0           — 卡通动画+风格化（备-漫剧）
     │   └─ Sora               — 世界模拟+超长连贯（待接入）
     │
     ├─ 🗣️ 音频/TTS组（配音/音乐/音效）
     │   ├─ 豆包TTS            — 多角色配音+情感语调（主）
     │   ├─ 通义千问TTS        — 多语言+朗读风格（备）
     │   ├─ Suno/AudioCraft    — BGM+音效生成（主）
     │   └─ ElevenLabs         — 超真实人声+多语言（备）
     │
     ├─ 🎭 数字人口播组（唇动+肢体）
     │   ├─ InfiniteTalk       — 唇同步1.8mm精度+全身体动（主-本地部署）
     │   ├─ HeyGen             — 数字人形象+口播生成（备-云API）
     │   ├─ 硅基智能           — 中文数字人+电商口播（备）
     │   └─ D-ID               — 照片驱动口播（轻量备）
     │
     └─ 🎞️ 剪辑合成组（成片输出）
         ├─ FFmpeg             — 核心合成引擎（后端）
         ├─ Remotion           — React驱动的程序化视频合成（前端预览）
         └─ 剪映SDK            — 模板化剪辑+特效（备）
```

### 4.2 模型选择策略

- **自动路由**：根据产品类型、风格、品质要求自动选择最优模型组合
- **用户可选**：高级用户可手动指定模型
- **备选降级**：主模型超时/失败时自动切换到备选模型
- **组合模式**：不同阶段使用不同模型（如 DeepSeek 写故事 + Seedance 生视频）

## 5. 前置质量保障系统（关键！）

> **这是即影产品的核心壁垒。** 在每一次生成调用之前，系统必须自动完成以下质量保障流程。不做完不允许进入生成阶段。

### 5.1 角色圣经系统（Character Bible System）

```
┌──────────────────────────────────────────────────┐
│                  角色圣经                          │
│                                                    │
│  【角色名】小明                                    │
│  【性别】男 | 【年龄】25岁 | 【体型】偏瘦          │
│  【发型】灰色短发，略微凌乱                        │
│  【上衣】破旧蓝色牛仔夹克，右肩有补丁              │
│  【下装】深灰色工装裤                              │
│  【配饰】银色吊坠项链 + 黑色手套                    │
│  【面部特征】坚毅眼神，右眉有细小疤痕              │
│  【典型表情】沉思时皱眉，战斗时咬牙                 │
│  【标志性姿势】左手插兜，右手摸项链                │
│                                                    │
│  角色定妆照（下一阶段生成）                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │  正面     │  │  侧面    │  │  背面    │        │
│  └──────────┘  └──────────┘  └──────────┘        │
│                                                    │
│  角色在不同情绪下的表现                            │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │  开心    │  │  悲伤    │  │  愤怒    │        │
│  └──────────┘  └──────────┘  └──────────┘        │
└──────────────────────────────────────────────────┘
```

**执行流程**：
1. 系统从用户描述中提取角色信息
2. 使用LLM自动补全完整的角色外形描述（结构化JSON）
3. 调用图像模型生成角色定妆照（正面+侧面+背面）
4. 角色圣经贯穿所有后续阶段的提示词

## 6. 图片产品线

### 6.1 爆款封面图

| 维度 | 说明 |
|------|------|
| 用户输入 | 一句话主题 + 可选参考图 + 平台选择（抖音/小红书/YouTube/B站） |
| 角色需求 | 如需人物，自动建角色圣经 |
| 提示词要求 | 主题突出，强视觉冲击，适合平台封面尺寸规范 |
| 输出 | 3-5张封面候选，含标题文字建议 + 点击率预测分 |
| 对标分析 | 可选：提供对标账号封面 → 分析爆款元素 → 风格复刻+差异化 |

### 6.2 海报设计

| 维度 | 说明 |
|------|------|
| 用户输入 | 一句话主题 + 可选参考风格 + 可选LOGO + 尺寸选择 |
| 场景设定 | 根据主题自动匹配场景 |
| 提示词要求 | 构图平衡，文字留白区域预留，品牌色融入 |
| 输出 | 2-3张海报设计稿，含排版建议 + 可编辑的PSD/AI文件（进阶） |
| 商品场景 | 支持产品图上传 → AI融入海报场景 |

### 6.3 商品详情页

| 维度 | 说明 |
|------|------|
| 用户输入 | 产品名称 + 一句话卖点 + 产品多角度图（上传） + LOGO + 价格 |
| 产品矩阵 | 必须上传正面/侧面/细节/使用场景至少3张参考图 |
| 详情页结构 | AI 自动规划：主图→痛点→卖点→使用场景→参数→评价→购买引导 |
| 输出 | 一组详情页图片（5-8张），适配电商平台尺寸 |
| 文案 | 自动生成卖点文案，可多版本A/B测试 |

### 6.4 LOGO/品牌设计

| 维度 | 说明 |
|------|------|
| 用户输入 | 品牌名 + 行业 + 关键词（如"科技感""亲和力"） + 参考风格 |
| 提示词要求 | 矢量风格，可缩放，色彩系统定义 |
| 输出 | 5-10个LOGO方案，含logo+应用场景mockup（名片/水印/招牌） |
| 迭代 | 用户选择偏好方向 → AI 细化生成 |

### 6.5 用户创意生图

| 维度 | 说明 |
|------|------|
| 用户输入 | 自由描述 + 风格选择 + 可选参考图 |
| 自由度 | 用户任意描述，不限主题 |
| 输出 | 1-4张高质感生成图 |

## 10. 自媒体创业赋能平台（商业闭环）

> **核心商业模式**: 20元落地一家线上自媒体公司
> **目标用户**: 想通过自媒体赚钱但不知从何下手的普通人（学生/宝妈/职场人/退休人群）
> **核心理念**: 思见做99%的幕后工作，用户只需做那1%的"发布"动作

### 10.2 为什么是「半自动化」？

> **行业背景**: 2025年起，抖音/小红书/视频号等平台严厉打击全自动化运营，机器批量发布会被识别封号。
>
> **思见策略**: 不碰红线。思见做所有"脑力工作"（选题、文案、视频、运营策略），
> 用户做"体力工作"（下载、打开APP、粘贴、发布）。
> 这恰好把用户体验从"看机器干活"升级为"我自己的自媒体生意"——用户的归属感和参与感反而更强。

#### 10.4.3 AI 智能客服系统

**功能**：自动回复用户在各大平台收到的评论、私信、咨询。

| 类型 | 处理方式 | 示例 |
|------|---------|------|
| 产品咨询 | 自动回复产品卖点+引导下单 | "这款我用了两周，最大的感受是……链接在主页哦～" |
| 价格询问 | 自动回复价格+限时优惠 | "现在刚好有活动，XX元起，比平时便宜XX呢！" |
| 合作邀约 | 自动留联系方式+引导私域 | "感谢关注！方便加个微信详聊吗？我的微信号是……" |
| 负面评论 | 礼貌回复+私信解决 | "抱歉让你有不好的体验，方便私信我详细说说吗？" |
| 普通互动 | 真诚回复+引导关注 | "谢谢喜欢！明天会更新XX主题，记得来看哦～" |

**技术实现**：
- 每个用户分配一个专属 AI 客服 Agent
- 客服 Agent 通过学习用户的 15 条以上回复样本，模仿其说话风格
- 所有回复需经过"安全合规过滤"（不碰政治/不承诺做不到的事）
- 用户可随时查看/编辑 AI 即将发出的回复
