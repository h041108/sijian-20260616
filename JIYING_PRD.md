## 1. 产品总览

### 1.1 一句话定义

> **即影 = 20元让一个人拥有一家王炸级自媒体公司**
> 15个AI专家级Agent + 1个智能调度大脑 + 每天30秒审核时间 = 全自动内容工厂

### 1.2 战略价值

```
思见平台 = 认知层（思维空间/思维线）              ← 理解用户怎么想
         ↓
         即影 = 创作层（Agent工厂）               ← AI替用户做出内容
         ↓
         商业层（自媒体创业平台）                  ← 帮用户赚钱
```

即影的核心竞争力不在"生成能力"（大家都在接入AI模型），而在**智能体调度系统**——15个专业Agent协同工作，每个Agent都精通一个垂直领域，主调度Agent负责拆解→分配→审核→汇总。

### 1.3 产品原则

| 原则 | 说明 |
|------|------|
| **人类审核，AI执行** | AI做99%的工作，用户做最后1%的审核+发布 |
| **专家级品质** | 每个Agent都是对应领域的"专家"，不是通用AI |
| **30秒体验** | 用户每天只需花30秒审核内容 |
| **渐进式介入** | 小白用全自动，高手用半自动，想单干的用单品模式 |
| **毛利94%+** | 单用户月成本¥4.2，定价¥99起 |

### 2.1 主调度Agent（Master Orchestrator）

这是整个系统的"大脑"，也是最核心的组件。

**职责**：
1. **意图理解**：接收用户输入的一句话/产品描述/品牌信息，理解用户到底要什么
2. **任务拆解**：把用户需求拆解为子任务，分配给对应的专业Agent
3. **上下文传递**：确保每个Agent获得所需的全部上下文（角色圣经、品牌定位、风格指南等）
4. **结果审核**：对每个Agent的输出做质量检查，不合格的打回重做
5. **冲突仲裁**：当Agent之间的输出有冲突时（如封面设计与人设不符），主调度判断取舍
6. **成品聚合**：把所有Agent的输出合成为最终交付物

**Prompt结构（简化）**：

```
你是即影平台的主调度Agent，负责协调15个专业Agent协同工作。

【当前用户档案】
- 行业：{niche}
- 品牌：{brand}
- 平台：{platform}
- 人设：{persona}

【今日任务】
- 类型：{daily_content | new_account_setup | strategy_adjust}
- 具体要求：{user_input}

【可用Agent列表】
- 策划群：Agent 0-品牌定位, Agent 1-商业策略, Agent 2-人设建模, ...
- 生产群：Agent 3-提示词大师, Agent 4-脚本分镜, ...
- 优化群：Agent 7-数据分析, Agent 8-投流分析, ...

请按以下流程工作：
1. 分析用户输入，明确任务类型和优先级
2. 拆解为子任务，分配给对应Agent（可并行/可串行）
3. 每个Agent输出后做质量检查
4. 汇总最终交付物
```

## 3. 15个Agent详解

### 3.1 策划群（策略层）— 先想清楚再做

这些Agent负责在动手创作之前，把"做什么"想清楚。

#### Agent 0：品牌定位专家

| 维度 | 说明 |
|------|------|
| 触发条件 | 新用户启动 / 品牌策略调整 |
| 输入 | 用户填写的行业、品牌名称、一句话描述 |
| 核心产出 | 品牌定位文档：定位语(1句话) + 核心价值(3点) + 目标受众 + 品牌人格 + 竞品定位图 |
| 输出格式 | 结构化文档 + 视觉指南（色板/字体/风格关键词） |
| 下游消费 | Agent 2人设建模、Agent 3提示词、图片工作室 |
| 用户审核 | ✅ 需要用户确认定位后才进入下一步 |

#### Agent 1：商业策略分析师（原E01）

| 维度 | 说明 |
|------|------|
| 触发条件 | 创始人IP模式 / 商业分析需求 |
| 输入 | Agent 0的品牌定位 + 用户商业目标 |
| 核心产出 | 创始人深度分析报告 + 可做短视频的IP方向 + 变现路径规划 |
| 输出格式 | 图文报告（可转为短视频脚本） |

#### Agent 2：人设建模专家（原E02）

| 维度 | 说明 |
|------|------|
| 触发条件 | 新账号启动 / 内容风格调整 |
| 输入 | Agent 0的品牌定位 + Agent 1的商业策略 |
| 核心产出 | 完整人设方案：人设标签(3-5个) × 语气风格 × 内容调性 × 视觉风格 × 口头禅/招牌动作 |
| 联系 | 人设输出直接影响图片工作室的"风格锚点"和漫剧引擎的"角色圣经" |

#### Agent 9：知识图谱专家（原E10）

| 维度 | 说明 |
|------|------|
| 触发条件 | 进入新行业 / 行业深度分析 |
| 输入 | 行业关键词 + 品牌定位 |
| 核心产出 | PESTEL分析 + SWOT分析 + 行业竞争格局图 + 关键趋势 + 机会窗口 |
| 输出格式 | 知识图谱（JSON）+ 可视化报告 |

#### Agent 13：选题分析师（原E15）

| 维度 | 说明 |
|------|------|
| 触发条件 | 每日内容生成 / 热点追踪 |
| 输入 | 人设方案 + 行业趋势 + 今日热点 |
| 核心产出 | 8维度×9风格标题方案：覆盖不同情绪钩子/内容形式/热点结合 |
| 输出 | 今日最佳选题TOP 3 + 各选题的切入角度建议 |

### 3.2 生产群（执行层）— 把方案做出来

#### Agent 3：提示词大师（原E04）

| 维度 | 说明 |
|------|------|
| 触发条件 | 图片/视频生成前 |
| 输入 | 策划群的全部输出 + 用户上传的参考图 |
| 核心产出 | 结构化prompt（适配Midjourney/即梦/Flux/DALL-E），含角色锚点+风格锚点+场景锚点 |
| 质量要求 | 每个prompt自动评分≥92分（内置质检），包含角色一致性校验 |
| 输出 | 一组优化后的多模型提示词，一键送至图片工作室 |

#### Agent 4：脚本分镜导演（原E05）

| 维度 | 说明 |
|------|------|
| 触发条件 | 视频/漫剧生成前 |
| 输入 | Agent 13的选题 + 人设方案 |
| 核心产出 | 含预算物料的执行级拍摄方案：分镜表 + 镜头卡 + 转场标注 + 时长分配 |
| 输出格式 | 分镜脚本JSON + 视觉参考图 |

#### Agent 5：BGM作曲家（原E06）

| 维度 | 说明 |
|------|------|
| 触发条件 | 视频生成前 |
| 输入 | 脚本情绪曲线 + 视频时长 + 风格参考 |
| 核心产出 | 情感匹配分析 + 推荐曲目（含BPM/情绪标签/推荐位置） |

#### Agent 6：音效设计师（原E07）

| 维度 | 说明 |
|------|------|
| 触发条件 | 视频生成前 |
| 输入 | 分镜脚本中的场景列表 |
| 核心产出 | 4层声音蓝图：环境音 + 动作音 + 情绪音 + 过渡音 |

#### Agent 12：封面灵感师（原E14）

| 维度 | 说明 |
|------|------|
| 触发条件 | 内容生成后（用于发布） |
| 输入 | 今日内容标题 + 核心要点 + 参考风格 |
| 核心产出 | 3套封面方案（每套含：视觉方案 + 标题排版 + 配色方案 + 点击率预测分） |

### 3.3 优化群（增长层）— 越做越好

#### Agent 7：数据分析师（原E08）

| 维度 | 说明 |
|------|------|
| 触发条件 | 发布后24小时（数据积累后） |
| 输入 | 各平台播放/互动/转化数据 |
| 核心产出 | 三维诊断报告：内容维度诊断(哪个选题好/差) + 时间维度诊断(什么时候发最好) + 用户维度诊断(粉丝画像变化) |

#### Agent 8：投流优化师（原E09）

| 维度 | 说明 |
|------|------|
| 触发条件 | 用户付费投流后 |
| 输入 | 投流数据 + 内容数据 |
| 核心产出 | ROI诊断 + 预算优化建议 + 定向人群建议 + 素材轮换策略 |

#### Agent 10：标题拆解师（原E11）

| 维度 | 说明 |
|------|------|
| 触发条件 | 内容发布前（标题优化） |
| 输入 | 今日内容 + 行业数据 |
| 核心产出 | 5维度评分（情绪/好奇心/具体度/关键词/长度） + 3版优化建议 |

#### Agent 11A：爆款复刻师（原E12）

| 维度 | 说明 |
|------|------|
| 触发条件 | 用户提供对标链接/文案 |
| 输入 | 对标爆款链接或文案 |
| 核心产出 | 基因解析（标题结构/钩子/话术/节奏/转化点） + 3版差异化复刻文案 |

#### Agent 11B：评论分析师（原E13）

| 维度 | 说明 |
|------|------|
| 触发条件 | 内容发布后有评论数据 |
| 输入 | 评论区数据 |

#### Agent 14：标签SEO专家

| 维度 | 说明 |
|------|------|

## 5. 漫剧生成引擎

### 5.1 什么是漫剧

> **漫剧 = 用漫画/插画风格讲述故事的视频形式**
> 图片序列 → 用户编排顺序 → 配音(TTS) → 运镜 → 字幕 → BGM → 封面 → 导出

### 5.2 完整流水线

```
用户提供/图片工作室生成一组图片
     ↓
┌────────────────────────────────────────────┐
│  步骤1：图片序列编排                        │
│  ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐ ┌──┐           │
│  │1 │→│2 │→│3 │→│4 │→│5 │→│6 │  ← 拖拽排序│
│  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘           │
```

## 7. 用户30秒审核模式

### 7.1 设计理念

> 用户不是创作者，是**审核者**。
> AI做出所有内容，用户只需要决定"发还是不发"。

### 7.2 每日流程

```

     ↓
打开思见 → 看到今日内容待审核列表
     ↓
┌─────────────────────────────────────────────────┐
│  今日待审核（3条图文 + 1条漫剧）                 │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │ 📝 图文1：{标题预览}                     │    │
│  │   文案预览（前50字）                     │    │
│  │   [✅ 确认] [✏️ 改两字] [🗑️ 跳过]       │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │ 📝 图文2：{标题预览}                     │    │
│  │   ...                                    │    │
│  │   [✅ 确认] [✏️ 改两字] [🗑️ 跳过]       │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  ┌─────────────────────────────────────────┐    │
│  │ 🎬 漫剧1：{标题}                         │    │
│  │   [▶️ 预览15秒]                          │    │
│  │   [✅ 确认] [✏️ 改两字] [🗑️ 跳过]       │    │
│  └─────────────────────────────────────────┘    │
│                                                 │
│  全部确认后 → [🚀 一键发布]                     │
└─────────────────────────────────────────────────┘
     ↓
{已发布} → AI自动追踪数据 → 明天优化
{已跳过} → AI分析跳过原因 → 下次避免类似内容
{已修改} → AI学习修改偏好 → 下次更贴近用户口味
```

### 7.3 审核交互详解

| 操作 | 含义 | AI行为 |
|------|------|--------|
| ✅ 确认 | 内容完美，直接发 | 记录"用户喜欢这个风格" |
| ✏️ 改两字 | 内容不错但要小改 | 弹出编辑框改完→发，记录修改偏好 |
| 🗑️ 跳过 | 这个不行，不发 | 不发布，AI分析原因，后续避免类似选题 |
| 不看直接发 | 信任AI，不审了 | 默认发布，记录"高度信任" |
