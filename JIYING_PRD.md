## 1. 产品总览

### 1.1 一句话定义

> **即影 = 思见平台的 AI 全媒体生成中枢**——用户提供一句话、一张参考图或一个参考文案，由全球顶级多模态大模型驱动，自动完成从"创意构思→提示词工程→角色/场景建立→多格式生成→质量校验→成品交付"的完整创作闭环。

### 1.2 战略价值

```
思见平台 = 认知层（思维空间/思维线）              ← 理解用户
         ↓
         即影 = 创作层（图片/视频/漫剧/短剧）     ← 创造内容
         ↓
         商业层（15引擎/数据运营/变现）           ← 实现价值
```

即影是思见从"认知工具"升级为"创作平台"的战略性产品。如果品质不够顶，整个平台的竞争力就不可持续。

### 1.3 产品原则

| 原则 | 说明 |
|------|------|
| **轻参与，重品质** | 用户一句话启动，AI 主导全流程。但用户可选择性提供参考素材来提升品质 |
| **前置做足，后顾无忧** | 生成之前的准备工作（提示词/角色/场景/参考图）比生成本身更重要 |
| **全球顶级引擎** | 每个环节调用最适合的大模型，不自我设限，持续接入最新SOTA模型 |
| **连续性第一** | 多帧/多镜头必须保证角色、风格、场景的绝对一致 |
| **成品为王** | 所有生成结果可下载、可分享、可商用 |

## 2. 产品形态矩阵

### 2.1 两大产品线

```
即影
├── 🖼️ 图片产品线（Image Products）
│   ├── 📱 爆款封面图          — 抖音/小红书/视频号封面
│   ├── 🎨 海报设计            — 活动海报/商品海报/品牌海报
│   ├── 📋 商品详情页          — 电商产品展示图/详情页
│   ├── 🏷️ LOGO/品牌设计       — LOGO生成+场景应用
│   ├── 👤 人物写真/形象照      — 数字人形象/IP角色定妆照
│   └── 🌄 用户创意生图        — 用户自由描述生成各种场景图
│
└── 🎬 视频产品线（Video Products）
    ├── 📱 爆款短视频          — 15-60s 抖音/快手/TikTok 热门视频
    ├── 🛒 带货视频            — 产品展示+卖点讲解+引导下单
    ├── 📚 漫剧 (Manga Anime)  — 漫画风格叙事动画
    ├── 🎭 短剧 (Short Drama)  — 多镜头剧情短片，6-12镜头
    ├── 📖 教育/知识讲解       — 知识科普+数字人讲解
    └── 🗣️ 口播/数字人播报     — 唇动同步+肢体动作的自然口播
```

| 比例 | 1:1 / 4:3 / 16:9 / 9:16 / 3:4 / 自定义 | 16:9 / 9:16 / 1:1 |
| 格式 | PNG / JPG / WebP | MP4 / MOV / WebM |
| 时长 | — | 15s / 30s / 60s / 90s / 自定义 |

## 3. 用户旅程

### 3.1 典型路径

```
用户进入即影
     │
     ├─ 选择产品类型: [图片] [视频]
     │
     ├─ 选择产品子类型: [封面] [海报] [详情页] [LOGO] ...
     │                    [短剧] [漫剧] [带货] [口播] [教育] ...
     │
     ├─ 提供核心输入（用户轻参与）
     │   ├─ 📝 一句话描述（必需）
     │   ├─ 🖼️ 参考图（可选 — 强烈建议提供）
     │   ├─ 📄 参考文案/链接（可选 — 爆款对标）
     │   ├─ 🏷️ LOGO/品牌素材（可选 — 涉及品牌时）
     │   └─ ⚙️ 高级参数（角色/风格/比例/时长等）
     │
     ├─ 【系统】自动执行前置质量保障（见第5章）
     │
     ├─ 【系统】调用AI引擎生成 → 用户实时预览进度
     │
     ├─ 用户审查结果
     │   ├─ ✅ 满意 → 下载/分享/发布
     │   └─ 🔄 不满意 → 局部修改/重新生成/调整参数
     │
     └─ 【系统】自动保存至作品集 + 数据追踪
```

### 3.2 用户参与度分级

| 层级 | 参与度 | 典型输入 | 品质预期 |
|------|--------|---------|---------|
| L1 | 极轻 | 一句话提示 | 80分 — 通用品质 |
| L2 | 轻度 | 一句话 + 1张参考图 | 85分 — 风格准确 |
| L3 | 适中 | 一句话 + 参考图 + 参考文案/对标链接 | 90分 — 爆款品质 |
| L4 | 深度 | L3 + 角色三视图 + 场景限定 + 产品多角度图 | 95分 — 商业级 |

## 4. 核心模型引擎

### 4.1 模型调度架构

```
即影模型路由
     │
     ├─ 📝 LLM/剧本组（故事/分镜/提示词）
     │   ├─ DeepSeek V3        — 中文故事+逻辑结构+分镜脚本（主）
     │   ├─ Claude 4           — 创意叙事+角色塑造+情感表达（备）
     │   └─ 通义千问/Qwen      — 中文适配+文化语境（备）
     │
     ├─ 🖼️ 图像生成组（图片/关键帧）
     │   ├─ Midjourney V7      — 艺术质感+光影构图（主-高端）
     │   ├─ 即梦/Jimeng        — 中式美学+角色一致性（主-高频）
     │   ├─ Flux/StableDiffusion3 — 快速迭代+开源可控（备）
     │   └─ DALL-E 3           — 精确指令跟随+文字渲染（备）
     │
     ├─ 🎬 视频生成组（动态镜头/动画）
     │   ├─ Seedance 2.0       — 文生视频+图生视频+动作流畅（主）
     │   ├─ 可灵/Kling         — 物理模拟+动作连贯（备）
     │   ├─ Runway Gen-4       — 电影级画质+运动模糊（备）
     │   ├─ Pika 2.0           — 卡通动画+风格化（备-漫剧）
     │   └─ Sora               — 世界模拟+超长连贯（待接入）
     │
     ├─ 🗣️ 音频/TTS组（配音/音乐/音效）
     │   ├─ 豆包TTS            — 多角色配音+情感语调（主）
     │   ├─ 通义千问TTS        — 多语言+朗读风格（备）
     │   ├─ Suno/AudioCraft    — BGM+音效生成（主）
     │   └─ ElevenLabs         — 超真实人声+多语言（备）
     │
     ├─ 🎭 数字人口播组（唇动+肢体）
     │   ├─ InfiniteTalk       — 唇同步1.8mm精度+全身体动（主-本地部署）
     │   ├─ HeyGen             — 数字人形象+口播生成（备-云API）
     │   ├─ 硅基智能           — 中文数字人+电商口播（备）
     │   └─ D-ID               — 照片驱动口播（轻量备）
     │
     └─ 🎞️ 剪辑合成组（成片输出）
         ├─ FFmpeg             — 核心合成引擎（后端）
         ├─ Remotion           — React驱动的程序化视频合成（前端预览）
         └─ 剪映SDK            — 模板化剪辑+特效（备）
```

### 4.2 模型选择策略

- **自动路由**：根据产品类型、风格、品质要求自动选择最优模型组合
- **用户可选**：高级用户可手动指定模型
- **备选降级**：主模型超时/失败时自动切换到备选模型
- **组合模式**：不同阶段使用不同模型（如 DeepSeek 写故事 + Seedance 生视频）

## 5. 前置质量保障系统（关键！）

> **这是即影产品的核心壁垒。** 在每一次生成调用之前，系统必须自动完成以下质量保障流程。不做完不允许进入生成阶段。

### 5.1 角色圣经系统（Character Bible System）

```
┌──────────────────────────────────────────────────┐
│                  角色圣经                          │
│                                                    │
│  【角色名】小明                                    │
│  【性别】男 | 【年龄】25岁 | 【体型】偏瘦          │
│  【发型】灰色短发，略微凌乱                        │
│  【上衣】破旧蓝色牛仔夹克，右肩有补丁              │
│  【下装】深灰色工装裤                              │
│  【配饰】银色吊坠项链 + 黑色手套                    │
│  【面部特征】坚毅眼神，右眉有细小疤痕              │
│  【典型表情】沉思时皱眉，战斗时咬牙                 │
│  【标志性姿势】左手插兜，右手摸项链                │
│                                                    │
│  角色定妆照（下一阶段生成）                        │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │  正面     │  │  侧面    │  │  背面    │        │
│  └──────────┘  └──────────┘  └──────────┘        │
│                                                    │
│  角色在不同情绪下的表现                            │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐        │
│  │  开心    │  │  悲伤    │  │  愤怒    │        │
│  └──────────┘  └──────────┘  └──────────┘        │
└──────────────────────────────────────────────────┘
```

**执行流程**：
1. 系统从用户描述中提取角色信息
2. 使用LLM自动补全完整的角色外形描述（结构化JSON）
3. 调用图像模型生成角色定妆照（正面+侧面+背面）
4. 角色圣经贯穿所有后续阶段的提示词

## 6. 图片产品线

### 6.1 爆款封面图

| 维度 | 说明 |
|------|------|
| 用户输入 | 一句话主题 + 可选参考图 + 平台选择（抖音/小红书/YouTube/B站） |
| 角色需求 | 如需人物，自动建角色圣经 |
| 提示词要求 | 主题突出，强视觉冲击，适合平台封面尺寸规范 |
| 输出 | 3-5张封面候选，含标题文字建议 + 点击率预测分 |
| 对标分析 | 可选：提供对标账号封面 → 分析爆款元素 → 风格复刻+差异化 |

### 6.2 海报设计

| 维度 | 说明 |
|------|------|
| 用户输入 | 一句话主题 + 可选参考风格 + 可选LOGO + 尺寸选择 |
| 场景设定 | 根据主题自动匹配场景 |
| 提示词要求 | 构图平衡，文字留白区域预留，品牌色融入 |
| 输出 | 2-3张海报设计稿，含排版建议 + 可编辑的PSD/AI文件（进阶） |
| 商品场景 | 支持产品图上传 → AI融入海报场景 |

### 6.3 商品详情页

| 维度 | 说明 |
|------|------|
| 用户输入 | 产品名称 + 一句话卖点 + 产品多角度图（上传） + LOGO + 价格 |
| 产品矩阵 | 必须上传正面/侧面/细节/使用场景至少3张参考图 |
| 详情页结构 | AI 自动规划：主图→痛点→卖点→使用场景→参数→评价→购买引导 |
| 输出 | 一组详情页图片（5-8张），适配电商平台尺寸 |
| 文案 | 自动生成卖点文案，可多版本A/B测试 |

### 6.4 LOGO/品牌设计

| 维度 | 说明 |
|------|------|
| 用户输入 | 品牌名 + 行业 + 关键词（如"科技感""亲和力"） + 参考风格 |
| 提示词要求 | 矢量风格，可缩放，色彩系统定义 |
| 输出 | 5-10个LOGO方案，含logo+应用场景mockup（名片/水印/招牌） |
| 迭代 | 用户选择偏好方向 → AI 细化生成 |

### 6.5 用户创意生图

| 维度 | 说明 |
|------|------|
| 用户输入 | 自由描述 + 风格选择 + 可选参考图 |
| 自由度 | 用户任意描述，不限主题 |
| 输出 | 1-4张高质感生成图 |

### 7.1 爆款短视频

| 阶段 | 内容 | 模型 | 备注 |
|------|------|------|------|
| ① 对标分析 | 用户提供爆款链接/文案 → AI拆解爆款公式 | DeepSeek | E03/E11引擎 |
| ② 差异化策略 | 基于对标生成差异化选题+脚本框架 | DeepSeek | E12引擎 |
| ③ 角色圣经 | 如有出镜角色，建立角色定妆照 | Midjourney/即梦 | — |
| ④ 场景设定 | 每个场景生成设定+参考图 | Midjourney/即梦 | — |
| ⑤ 分镜脚本 | 按爆款节奏生成N镜头分镜（含转场衔接） | DeepSeek | 含视觉连续性标注 |
| ⑥ 提示词工程 | 为每个镜头生成多模型适配提示词 | DeepSeek | 含角色/场景锚点 |
| ⑦ 视觉生成 | 逐镜头生成关键帧 + 视频片段 | 即梦→Seedance | img2img 保证连续 |
| ⑧ BGM+配音 | 根据情绪曲线生成BGM + TTS配音 | Suno+豆包TTS | 含音素时间戳 |
| ⑨ 合成 | 视频+音频+字幕+转场 → 成片 | FFmpeg/Remotion | — |

### 7.2 带货视频

| 特点 | 说明 |
|------|------|
| 输入要求 | 产品多角度图（必传） + LOGO（必传） + 核心卖点 + 价格 |
| 脚本结构 | 15-30s：痛点→产品出现→卖点展示→使用场景→价格→引导下单 |
| 视觉要求 | 产品360°展示，使用场景真实，卖点文字标注 |
| 数字人 | 可选数字人口播讲解，自动唇动同步 |
| 输出 | 3个版本：完整版(30s) / 精简版(15s) / 纯产品展示版(10s) |

### 7.3 漫剧

| 特点 | 说明 |
|------|------|
| 风格 | 日系动漫 / 国风水墨 / 美式卡通 / 赛博朋克 / 皮克斯3D |
| 脚本 | 从一句话故事 → 完整漫画分镜 |
| 角色 | 每个角色必须有圣经+定妆照 |
| 场景 | 每个场景独立设定+参考帧 |

| 音乐 | 场景适配BGM + 音效 |

### 7.4 短剧

| 特点 | 说明 |
|------|------|
| 长度 | 60-180秒 (6-12镜头) |
| 结构 | 起承转合，标配：钩子(3s) → 展开(40%) → 冲突(30%) → 高潮(20%) → 结局(7%) |
| 角色要求 | 必须：每个角色有圣经+正面/侧面/背面定妆照 |
| 镜头连续性 | **强制要求**：每两个相邻镜头要有 30%+ 的视觉重叠元素 |
| 转场标注 | 每个镜头标注：入动作 / 出动作 / 转场类型 |
| 对话 | 自动标注音素时间轴 → 数字人/角色唇动同步 |
| 品质门槛 | 前置质检得分 < 85 不允许进入生成 |

### 7.5 教育/知识讲解

| 特点 | 说明 |
|------|------|
| 形式 | 数字人口播 + PPT/图示切换 |
| 脚本 | 知识点结构化 + 类比/案例穿插 |
| 视觉 | 自动生成图示 + 文字标注 + 动画效果 |
| 数字人 | 可选形象，唇动同步+手势配合讲解节奏 |
| 时长 | 60-300秒 |

### 7.6 口播/数字人播报

| 特点 | 说明 |
|------|------|
| 技术栈 | InfiniteTalk (本地) / HeyGen (云端) |
| 唇动同步 | TTS输出带音素时间戳 → 数字人逐帧唇形匹配 |
| 肢体动作 | 根据文本情感自动生成手势/头部动作/身体姿态 |
| 表情 | 文本情绪 → 面部表情驱动（微笑/严肃/惊讶等） |
| 背景 | AI 生成/用户自选，支持绿幕替换 |
| 音画同步 | 强制：音频轨道时间轴与视频帧严格对应 |

### 8.2 提示词工程系统（Prompt Engineering System）

**这是即影的品质核心**——每个产品在生成前，必须经过完整的提示词工程流程：

```
角色圣经（JSON结构化角色描述）
     +
场景设定（每个场景的光线/色调/氛围）
     +
参考图分析（风格/色调/构图提取）
     +
产品参数（卖点/价格/LOGO/多角度图）
     +
对标分析（爆款脚本结构/话术/节奏）
     +
风格锚定（统一的艺术风格关键词）
     ↓
提示词聚合引擎
     ↓
┌─────────────────────────────────────────────┐
│  为每个镜头/每张图生成独立提示词              │
│                                              │
│  镜头1:                                      │
│  [即梦] 写实电影风格，一位灰发少年...         │
│  [Midjourney] A cinematic shot of...         │
│  [Seedance] ...                              │
│                                              │
│  镜头2:                                      │
│  [即梦] 延续镜头1的视觉风格，灰发少年...     │
│  [Midjourney] Same character as shot1...     │
│  [Seedance] ...                              │
│                                              │
│  ↓ 每个镜头提示词包含：                       │
│  1. 角色锚点（外表描述完全一致）              │
│  2. 场景锚点（场景设定参数）                  │
│  3. 风格锚点（统一风格关键词）                │
│  4. 转场锚点（前一镜头结束→本镜头开始）      │
│  5. 光照锚点（统一光照方向/色调）             │
└─────────────────────────────────────────────┘
```

## 10. 数据架构

### 10.1 核心数据结构（简略）

```typescript
// ── 角色圣经 ──
interface CharacterBible {
  id: string
  name: string
  gender: string
  age: string
  bodyType: string
  hairStyle: string
  top: string
  bottom: string
  accessories: string[]
  facialFeatures: string
  typicalExpressions: Record<string, string>
  signaturePose: string
  referenceImages: {
    front: string    // 正面定妆照 URL
    side: string     // 侧面定妆照 URL
    back: string     // 背面定妆照 URL
    emotions: Record<string, string>  // 不同情绪表情图
  }
  characterSeed: number   // 角色种子，用于保持一致性
}

// ── 场景设定 ──
interface SceneSetting {
  id: string
  sceneNumber: number
  name: string
  timeOfDay: string
  lighting: string
  weather: string
  colorPalette: string[]
  atmosphere: string
  restrictions: string[]
  referenceImage?: string
}

// ── 镜头连续性标注 ──
interface ShotContinuity {
  enterAction: string     // 入动作（如"从左走进画面"）
  exitAction: string      // 出动作（如"转身离开"）
  transition: "cut" | "fade" | "dissolve" | "wipe" | "match_cut"
  visualOverlap: string   // 与前一镜头的视觉重叠元素描述
  overlapRatio: number    // 重叠百分比（要求≥30%）
  characterPositions: Record<string, { x: number; y: number }>  // 各角色在画面中的位置
  continuityWithPrev: string  // 延续上一镜头的哪些视觉元素
}

// ── 产品参考矩阵 ──
interface ProductReference {
  productName: string
  sellingPoints: string[]
  price: string
  logo?: string
  images: {
    front: string
    side: string
    detail: string[]
    usageScene: string[]
  }
  benchmarkLinks?: string[]
  brandColors?: string[]
}

// ── 即影项目 ──
interface JiyingProject {
  id: string
  type: "image" | "video"
  subType: string           // "poster" | "cover" | "detail" | "short_drama" | ...
  userInput: string         // 用户一句话
  status: "draft" | "preparing" | "generating" | "review" | "completed" | "failed"
  
  // 参考素材
  referenceImages: string[]
  referenceAudio?: string
  referenceVideo?: string
  referenceCopy?: string
  benchmarkLinks?: string[]
  
  // 质量前置数据
  characterBibles?: CharacterBible[]
  sceneSettings?: SceneSetting[]
  productReference?: ProductReference
  
  // 生成结果
  prompts: PromptCard[]      // 所有镜头/图片的提示词
  results: GenerationResult[]
  
  // 元数据
  createdAt: string
  completedAt?: string
  modelChain: string[]       // 使用了哪些模型
  qualityScore: number       // 前置质量得分
}
```
