# 思见 2.0 完整升级方案

## 零、产品架构总览：孪生智能体

思见不是一个产品，是**两个智能体的协同系统**：

```
                         ┌──────────────────────────┐
                         │   思见 · 认知操作系统      │
                         │   Sijian Cognitive OS      │
                         └──────────┬───────────────┘
                                    │
              ┌─────────────────────┴─────────────────────┐
              │                                           │
    ┌─────────┴─────────┐                   ┌─────────────┴─────────────┐
    │   🧠 思见 Gemini   │                   │   🎬 即影 Gemini          │
    │   "帮你想清楚"      │                   │   "帮你做出来"             │
    ├───────────────────┤                   ├───────────────────────────┤
    │ 解决：              │                   │ 解决：                      │
    │ 我不知道自己在想什么  │                   │ 我有想法但做不出来            │
    │ 我看不到自己的盲点    │                   │ 我做了但质量不行             │
    │ 我的决策质量差       │                   │ 我做了但没人看              │
    │ 我不会深度思考       │                   │ 我做不出爆款               │
    │ 我积累的认知在流失    │                   │ 我的内容无法变现            │
    ├───────────────────┤                   ├───────────────────────────┤
    │ 输入：碎片/语音/截图  │                   │ 输入：一句话/脚本/爆款参考    │
    │ 输出：洞察/决策/日报  │                   │ 输出：视频/落地页/广告素材    │
    │ 用户：舵手（2分钟/天）│                   │ 用户：导演（点一下，看结果）   │
    │ 模式：异步·自主·后台  │                   │ 模式：全自动·端到端·直接可用  │
    └───────────────────┘                   └───────────────────────────┘
              │                                           │
              └─────────────────────┬─────────────────────┘
                                    │
                          ┌─────────┴─────────┐
                          │   共享数据层        │
                          │   思维指纹 + 创作DNA │
                          └───────────────────┘
```

**两个智能体共享同一个底层能力**：
- 61条思维线引擎（`thinking-lines.ts`）
- 四层认知分析（`cognition.ts` L1-L4）
- DeepSeek 推理层
- 搜索（Tavily/Serper）
- 多模型路由（`orchestrator.ts`）

**但面向不同的问题域**：
- 思见面向**认知质量**——输入碎片，输出洞察
- 即影面向**创作质量**——输入创意，输出作品

## 一、思见 Gemini 升级方案

### 1.1 当前状态 vs 目标状态

| 维度 | 当前（1.0） | 目标（2.0） |
|------|------------|------------|
| 用户角色 | 建筑师（手动搭建思维空间） | 舵手（审核日报 + 调方向） |
| 输入方式 | 在聊天框打字 | 微信转发/语音/截图/邮件/浏览器扩展 |
| 交互频率 | 用户主动打开才用 | AI 在后台持续工作，每日推送日报 |
| 核心价值 | 2D/3D 思维空间可视化 | **认知处理自动化**：碎片→结构→洞察 |
| 留存机制 | 用户记得回来用 | 日报推送 → 用户回来看 |
| 数据资产 | 独立的思维空间 | **思维指纹**：61维向量 × 时间序列 |
| 框架数量 | 11 种视觉框架 | 11 种 + 自动框架推荐 |
| 多模型 | DeepSeek 单模型 | DeepSeek + 多模型路由已就绪 |

### 1.2 升级路线图

#### Phase 1：日报闭环（2周）

**目标**：验证"用户愿意每天看 AI 整理的日报"

**实现**：

1. **`/api/daily-report/generate`** — 夜间批处理端点
   - 输入：用户过去 24 小时的聊天记录、思维空间操作
   - 处理：LLM 合并碎片 → 去重 → 识别模式 → 检测矛盾 → 发现盲点
   - 输出：结构化日报 JSON

2. **`/api/daily-report/send`** — 推送端点
   - Email 推送（nodemailer）
   - 浏览器 Notification API（已打开时）
   - 未来：微信公众号模板消息

3. **日报内容模板**：
```
📥 昨日归档：N 条碎片 → M 个思维节点
🔗 新建连接：A ↔ B、C ↔ D
⚠️ 矛盾发现：你周一认为 X，周三又说 Y
🕳️ 盲点提醒：你讨论了 A 13次，但从没提过 B
📊 思维趋势：注意力从 X 向 Y 转移
💡 今日建议：基于昨天的思考，今天可以深挖 Z
```

4. **前端改动**：`ChatPanel` 增加一个"今日日报"入口

#### Phase 2：多通道输入（3周）

**目标**：让用户"什么都不做"就能积累认知资产

**实现**：

1. **微信公众号接入**
   - 新文件：`src/app/api/wechat/webhook/route.ts`
   - 接收：文本消息、图片消息、语音消息、转发链接
   - 处理：图片 → OCR+理解 → 存入思维碎片池
   - 语音 → 转文字 → 存入
   - 链接 → 抓取内容 → 摘要 → 存入

2. **浏览器扩展**（轻量版，一个 manifest.json + content script）
   - 选中网页文字 → 右键"发送到思见"
   - 自动记录当前页面 URL 和标题

3. **碎片池 API**
   - 新文件：`src/app/api/fragments/route.ts`
   - GET/POST/DELETE 碎片
   - 每个碎片带 `source`（wechat/voice/screenshot/web/manual）
   - 新文件：`src/lib/fragment-pool.ts` — 碎片管理引擎

4. **自动标注**
   - 碎片到达时自动调用 `cognition.ts` 标注：思维线、情绪、认知负荷
   - 存入 localStorage + 未来 Supabase

#### Phase 3：思维指纹产品化（4周）

**目标**：把现有 `ThinkingMirror` 接口变成用户可见的产品

**实现**：

1. **思维指纹 Dashboard**
   - 新组件：`ThinkingFingerprintDashboard.tsx`
   - 展示：主导思维线、状态分布、意图分布、情绪历史
   - 时间维度：本周 / 本月 / 全部
   - 对比："和上周的你相比，你的因果思维提升了 15%"

2. **思维趋势预警**
   - "你连续 3 天认知负荷偏高，建议休息"
   - "你的发散思维占比下降到 10%，可能进入了思维固化"
   - "你本周的决策类思考减少了 40%，是不是在回避某个决定？"

3. **思维模式库**（匿名）
   - 收集用户思维空间拓扑结构（仅结构，去内容）
   - 新用户构建思维空间时，推荐相似模式
   - "你的思考结构跟高效决策者群体很像——他们习惯先发散再收敛"

#### Phase 4：舵手模式（6周）

**目标**：让 AI 自主构建思维空间，用户只管审核

**实现**：

1. **AI 自主构建**
   - 用户在聊天/微信/语音里说了什么 → AI 自动提取关键概念 → 创建节点 → 建立连接 → 选择框架
   - 用户看到的是"已经搭好的思维空间草稿"

2. **用户交互**
   - 👍 点赞 = 确认，保持
   - ✏️ 改字 = 修正，AI 学习
   - 🗑️ 删 = 不需要，AI 记录偏好
   - "方向偏了" = 重新构建

3. **学习闭环**
   - 用户的每次修正都被记录
   - `src/lib/experiments.ts` 的 A/B 框架已就绪，直接用于收集修正数据
   - 长期：AI 越来越懂这个用户的思维偏好

## 二、即影 Gemini 升级方案

### 2.1 当前状态 vs 目标状态

| 维度 | 当前 | 目标 |
|------|------|------|
| 视频类型 | 单镜头 | **多镜头完整短片**（已修复，待验证） |
| TTS 配音 | 无 | 火山引擎 TTS + 浏览器兜底（已有） |
| 背景音乐 | 无 | 免费音乐库 + AI 推荐 |
| 字幕 | 无 | DeepSeek 自动生成时间轴字幕 |
| 封面 | 即梦生成的图 | **AI 自动设计视频封面**（标题叠加+优化） |
| 发布渠道 | 无 | 下载 .webm（已有） + 一键发布草稿 |
| 数据反馈 | 无 | 发布后追踪数据回传 |
| 持续创作 | 每次都从头开始 | **模板复用 + 系列化创作** |
| 变现 | 未涉及 | 视频 → 带货落地页（计划中） |

### 2.2 升级路线图

#### Phase 1：完整短片输出（1周）

**当前状态**：parseShotsFromScript 已修复，生产待验证
**目标**：验证多镜头 → 多图 → Canvas 合成完整短片

- 修复 `runFullPipeline` 中 shot 解析使用 `script_breakdown` 输出
- 验证 `final_assembly` 产生完整时间轴
- 验证 Canvas 合成多镜头 `.webm`

#### Phase 2：配音 + 字幕 + BGM（2周）

1. **TTS 配音** — 已有 `/api/video/tts`（火山引擎 + 浏览器兜底）
   - 集成到 `visual_generation`：从分镜脚本提取对白/旁白 → 调用 TTS → 生成每镜头音频
   - 视频合成时合并音轨

2. **字幕生成**
   - 新文件：`src/lib/subtitle-generator.ts`
   - 从分镜脚本的对白/旁白字段提取文本
   - 生成 SRT/VTT 格式时间轴
   - Canvas 叠加字幕到视频帧

3. **BGM 推荐**
   - 新文件：`src/lib/bgm-recommender.ts`
   - 基于视频情绪曲线推荐音乐风格
   - 使用免版权音乐（Pixabay Music API 或本地库）
   - 或：直接调用火山引擎 TTS 生成背景音

#### Phase 3：视频封面 + 标题优化（1周）

1. **封面生成**
   - 从关键帧中选择情绪高潮帧 → 叠加标题文字 → 生成竖版/横版封面
   - 新端点：`/api/video/cover`

2. **标题优化**
   - 基于爆款模板的关键词 + 视频内容 → DeepSeek 生成 5 个备选标题
   - 每个标题带"预估点击率"评分

#### Phase 4：系列化创作（2周）

1. **模板保存**
   - 用户完成一个作品后，保存整个创作配置（风格、分镜结构、时长、画幅比）为模板
   - 下次创作时选择模板 → 只替换内容

2. **系列管理**
   - 多个作品组成系列（如"每日一个护肤技巧"）
   - 统一的视觉风格 + 片头片尾

3. **批量生成**
   - 输入 N 个主题 → 自动生成 N 个视频
   - 适合知识类/教程类账号

#### Phase 5：变现闭环（4周）

1. **视频 + 带货落地页**
   - 视频生成完后，自动生成一个简洁的带货页面
   - 包含：视频嵌入、产品图片、价格、购买按钮（Stripe）
   - 新端点：`/api/store/landing-page`

2. **分发草稿**
   - 生成"发布到抖音/视频号/TikTok"的内容包
   - 包含：视频文件 + 标题 + 描述 + 标签 + 封面

## 三、共享基础设施升级

两个 Gemini 共用的底层能力升级：

### 3.1 异步工作引擎

**新文件：`src/lib/async-worker.ts`**

```
核心设计：
- 用户离线后，后台继续运行批处理任务
- 任务队列：fragment_process / daily_report / trend_scan / blind_spot_check
- 每个任务有优先级 + 执行时间窗口
- 用户下次打开时，增量同步结果
```

**实现方式**：
- 浏览器端：Service Worker + Background Sync（离线消息队列）
- 服务端：Vercel Cron Jobs（定时触发批处理）
- 未来：独立 Worker 进程

### 3.2 统一通知中心

**新文件：`src/lib/notifications.ts`**
**新组件：`NotificationCenter.tsx`**

```
通知类型：
- 日报就绪
- 思维盲点发现
- 即影作品完成
- 视频发布数据回传
- 系统更新

渠道：
- 浏览器 Notification
- Email（nodemailer）
- 微信模板消息（未来）
```

### 3.3 统一数据层

**当前状态**：所有数据在 localStorage
**目标**：localStorage（主） + Supabase（可选同步）

```
数据模型：
- fragments（思维碎片）
- daily_reports（日报记录）
- thinking_fingerprint（思维指纹快照）
- video_templates（创作模板）
- video_series（作品系列）
- user_preferences（用户偏好）
```

保持"永不强制云同步"原则——localStorage 是主数据源，云端是可选的备份/同步。

## 四、商业模型升级

### 4.1 三层金字塔

| 层级 | 用户 | 定价 | 核心价值 | 收入占比 |
|------|------|------|---------|---------|
| **免费** | 所有人 | $0 | 日报 + 基础思维空间 + 输入/日限制 | 获客 |
| **Pro** | 深度用户 | ¥19/月 | 无限输入 + 深挖模式 + 思维简报 + 即影基础 | 40% |
| **即影 Pro** | 创作者 | ¥29/月 | 无限视频生成 + 配音/BGM + 封面 + 模板 | 30% |
| **企业** | 机构 | ¥299-999/月 | 思维指纹 + 招聘辅助 + 培训诊断 + 即影批量 | 30% |

### 4.2 从订阅到分润

即影 Pro 用户生成带货视频 → 视频自带思见落地页 → 观众购买 → 思见分润 10-15%。

**这是 Polsia 想做但做不到的**——它的落地页是 AI Slop，思见的落地页基于认知质检通过的优质视频。

## 七、一句话总结

> **思见 2.0 = 两个孪生智能体，一个认知操作系统**
>
> - **思见 Gemini**（左脑）：帮你想清楚 — 碎片→日报→思维指纹→决策辅助
> - **即影 Gemini**（右脑）：帮你做出来 — 创意→视频→配音→封面→变现
>
> 它们共享同一套认知引擎，面向不同的人类痛点：
> **思见解决"思维质量"，即影解决"创作质量"。**
>
> Polsia 试图跳过思考直接执行，结果 93.7% 失败。
> 思见的路径是：**先确保想清楚了，再确保做出来了。**
