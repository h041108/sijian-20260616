> 核心理念：AI 辅助创作故事板，用户参与把控，视频生成作为输出选项。

## 三、四个功能的差异化设计

### 3.1 微短剧

| 项目 | 内容 |
|------|------|
| 默认时长 | 90 秒 |
| 默认镜头数 | 6 个 |
| 默认风格 | 写实风格 |
| 叙事结构 | 起承转合（四段式） |
| 默认比例 | 9:16（竖屏） |
| 适用场景 | 抖音/快手剧情号 |

**AI 写作差异：** 故事创世阶段使用"短剧"提示词模板，强调角色冲突、对话推进、情节反转。

**故事板特殊显示：** 每个镜头标注"起/承/转/合"所属章节。

### 3.2 漫剧

| 项目 | 内容 |
|------|------|
| 默认时长 | 120 秒 |
| 默认镜头数 | 8 个 |
| 默认风格 | 日系动漫 |
| 叙事结构 | 三幕式 |
| 默认比例 | 9:16 / 16:9 |
| 适用场景 | B站/小红书动漫区 |

**AI 写作差异：** 分镜提示词要求"画面描述带漫画感"，镜头语言突出视觉冲击力。

**故事板特殊显示：** 标注"分镜编号 + 漫画对话框文本"，支持添加拟声词。

### 3.3 知识讲解

| 项目 | 内容 |
|------|------|
| 默认时长 | 60 秒 |
| 默认镜头数 | 5 个 |
| 默认风格 | 扁平设计/白板风格 |
| 叙事结构 | 问题→拆解→结论 |
| 默认比例 | 16:9（横屏） |
| 适用场景 | 知识分享/教育科普 |

**AI 写作差异：** 脚本以"问题提出→概念解释→举例说明→总结"逻辑展开，强调信息密度。

**故事板特殊显示：** 每个镜头可添加"文字标注/数据点"，支持图表生成。

### 3.4 产品广告

| 项目 | 内容 |
|------|------|
| 默认时长 | 30 秒 |
| 默认镜头数 | 5 个 |
| 默认风格 | 高端质感/科技感 |
| 叙事结构 | 痛点→方案→使用→效果→行动 |
| 默认比例 | 9:16 / 16:9 |
| 适用场景 | 抖音带货/电商商品卡 |

**AI 写作差异：** 分镜强调产品特写、使用场景、转化话术。

**故事板特殊显示：** 每个镜头可添加"卖点标签"和"CTA（行动号召）文本"。

## 九、产品广告 —— 产品图上传播放

### 9.1 设计说明

产品广告不需要"角色"，需要的是**产品实拍图**。用户上传产品照片后，所有镜头的画面都以产品参考图为基础生成，产品外观在所有镜头中完全一致。

### 9.2 产品上传面板

```
┌──────────────────────────────────────────────┐
│  📦 产品素材                                 │
│                                               │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐     │
│  │ 正面  │  │ 侧面  │  │ 背面  │  │ 场景  │     │
│  │ img   │  │ img   │  │ img   │  │ img   │     │
│  │ [✕]   │  │ [✕]   │  │ [✕]   │  │ [✕]   │     │
│  └──────┘  └──────┘  └──────┘  └──────┘     │
│                                               │
│  产品名称: [智能扫地机器人]                     │
│  核心卖点: [自动集尘·激光导航·APP控制]          │
│                                               │
│  [🎬 一键生成广告]                              │
└──────────────────────────────────────────────┘
```

### 9.3 技术实现

| 环节 | 能力 | 说明 |
|------|------|------|
| 上传产品图 | ✅ Seedance 支持 `referenceImageUrls` | 直接传产品图做参考 |
| 产品外观一致 | ✅ **产品不会变** | 所有镜头用同一张产品参考图 |
| 背景/场景变化 | ✅ Seedream img2img + `image_strength` | 低强度保留产品，背景随 prompt 变化 |
| 多角度展示 | ⚠️ 需上传不同角度 | 产品各角度需用户提供 |

### 9.4 产品广告 vs 角色类视频的关键差异

| 维度 | 角色类（短剧/漫剧） | 产品类（广告） |
|------|-------------------|---------------|
| 一致性难度 | 🔴 角色长相无法100%保证 | 🟢 产品图直接参考，完全一致 |
| 参考图 | 需AI生成的虚拟形象 | 用户上传的产品实拍图 |
| 场景变化 | 多场景故事推进 | 围绕产品展示，场景为辅 |
| 用户参与 | 描述角色→看结果 | 上传产品→看结果 |

```

       │
       ▼

       │
       ▼
DeepSeek 拆解：钩子、节奏、结构、情绪曲线、转化话术
       │
       ▼

       │
       ▼

       │
       ▼

```

## 六、技术实现方案

### 6.1 现有技术可满足的部分

| 功能 | 实现方式 | 现有代码 |
|------|----------|----------|
| AI 写故事 → 分镜 | DeepSeek API + 6阶段流水线 | `video-factory.ts` `PIPELINE_STAGES` |
| 每阶段可编辑 | 故事板编辑组件 | `StoryboardEditor.tsx`（刚完成） |
| 角色模板创建 | 文字描述→Seedream 4.5→4选1 | `CharacterCreator.tsx`（刚完成） |
| 角色参考图入参 | `referenceImageUrls` 传 Seedance | Seedance API 第 44-46 行 |
| 单镜关键帧生成 | Seedream 4.5 图生图 | `/api/video/frame` + `image_strength` |
| 单段视频生成 | Seedance 图生视频 | `/api/video/seedance` |
| 口述转分镜 | 语音识别 + DeepSeek 拆解 | `voice-video.ts` |
| 风格预设（4 种） | `genre` 参数 | `video-factory.ts` `GENRE_PRESETS` |
| 角色模板存储 | localStorage + Supabase | `character-engine.ts` |

### 6.2 需要新增的技术能力

| 功能 | 方案 | 工作量 |
|------|------|--------|
| 故事板→最终拼接视频 | 客户端 Canvas 合成，返回可预览/下载 | 已有骨架，需完善 UI |
| 故事大纲可编辑 | AI 生成后显示为文本框，用户改完重新生成分镜 | 1 天 |
| 角色选中后注入流水线 | 在 `executeStage` 中将角色描述加入 prompt | 0.5 天 |
| 多角色简单支持 | 提供"主角""配角"两个槽位，各传一张参考图 | 1 天 |
| 场景模板（咖啡馆/教室等） | 预设 10-20 个场景描述文本，可选 | 1 天 |

### 6.3 明确做不到的（需坦诚告知用户）

| 做不到 | 原因 |
|--------|------|
| 多段视频间角色长相100%一致 | 全球无商用 API 能做到 |
| 多角色同框时各自保持身份 | API 不支持多参考图分离控制 |
| 镜头间的场景完全连续 | AI 不理解"同一场景"概念 |
| 精确控制角色表情/动作 | 只能通过 prompt 描述，不可靠 |
| 生成后局部修改（换装） | 视频无法局部重绘 |

### 6.4 改进建议：价值定位调整

> ~~AI 自动生成完整短剧~~ → **AI 辅助创建故事板 + 单段视频素材输出**

**对用户实际的交付物：**
1. ✅ 一份完整的分镜头脚本（可直接用于拍摄）
2. ✅ 每个镜头的关键帧预览图（辅助视觉参考）
3. ✅ 视频片段素材（每个镜头一段视频，可剪辑使用）
4. ✅ 可编辑、可修改、可迭代
