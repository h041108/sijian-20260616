# 即影 · 架构改进优先级表

> 基于 OpenMontage/Remotion 借鉴 + 即影现有代码基础
> 按"投入产出比"排序，从最快能见效的开始

---

## 🔴 P0 — 目前最大的功能缺口（1-3 天/项）

| # | 改进项 | 现在的状态 | 目标 | 借鉴来源 | 利用现有代码 | 工作量 |
|---|--------|-----------|------|----------|------------|--------|
| **1** | **粘贴竞品链接→自动拆解→生成** | 爆款拆解 API 存在但入口隐藏 | 每日内容引擎+影片工厂首页加一个输入框，粘贴小红书/抖音链接，自动拆解后注入流水线 | OpenMontage ref-driven creation | ✅ `/api/trends/deconstruct` 已有<br>✅ `viralTemplate` 注入逻辑已有<br>❌ 缺前端入口 | **0.5 天** |
| **2** | **Remotion 合成最终视频** | 最终合成输出 JSON 元数据，用户拿不到 MP4 | Remotion 读取各镜关键帧+时间轴，合成 MP4，用户可下载 | Remotion（MIT） | ❌ 需接入 Remotion<br>✅ 故事板数据已有 | **3 天** |
| **3** | **知识讲解→HTML/动画渲染** | 知识讲解模式只能生成图片和文案 | 用 React 组件直接渲染"信息图+文字+图表"动画，输出 MP4，不走 Seedance | OpenMontage HyperFrames | ✅ React 技术栈已有<br>❌ 动画组件需新建 | **2 天** |
| **4** | **一键全自动流水线** | 6 个阶段需要用户逐一点"运行" | 一个"🚀 全自动生成"按钮按顺序执行所有阶段，自动轮询 Seedance 结果 | OpenMontage agent-first pipeline | ✅ `executeStage` 逻辑已有<br>❌ 缺串联逻辑 | **1 天** |

## 建议执行顺序

```
第 1 周（最快见效）
  1 → 粘贴竞品链接  (0.5天)
  4 → 一键全自动流水线 (1天)
  5 → 免费图库配图 (1天)
  8 → Seedance 轮询进度 (1天)
  ── 用户此刻能完整走通"粘贴链接→自动生成→下载"全流程 ──

第 2 周（交付 MP4 视频）
  2 → Remotion 合成视频 (3天)
  7 → 转场+字幕 (2天)
  ── 用户拿到真实 MP4，不是 JSON ──

第 3 周（体验飞跃）
  3 → 知识讲解 HTML渲染 (2天)
  6 → Agent 导演模式 (1天)
  11 → 批量重生成 (1天)
  ── 生成速度和质量明显提升 ──

第 4 周（商业化）
  9 → 多渲染引擎 (3天)
  10 → 成本控制 (2天)
  12 → 导出 PR/剪映 (3天)
  ── 产品达到可商用状态 ──
```
